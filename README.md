## 브랜치 운영 규칙

- `master`: 실제 데모 배포용입니다. `dev`에서 검수가 끝난 뒤에만 올립니다.
- `dev`: 팀 공유 및 통합 검수용입니다. 기능 브랜치에서 충분히 확인한 뒤 merge합니다.
- `feature/*`: 기능이나 UX 실험 단위로 생성합니다. 작업 목적이 끝나면 `dev` 반영 여부를 위 표에 업데이트합니다.
- 새 작업을 시작할 때는 `dev`에서 새 브랜치를 파고, 작업이 끝나면 README의 브랜치 현황도 같이 갱신합니다.

## 최근 반영 내용

- 위치 오인 시 `다른 매장이에요`를 통해 인접 매장 후보로 바꿀 수 있습니다.
- 위치 후보를 고른 뒤에는 `이 매장으로 보기`를 눌러야 추천이 바뀝니다.
- 혜택 확인 화면은 하단 시트로 열리고, 필요한 카드/멤버십/예상 혜택/결제 버튼을 먼저 보여줍니다.
- 추천 기준은 `혜택`, `마일리지`, `실적`입니다.
- 추천 기준을 바꾸면 카드/혜택 변화를 먼저 보여주고, `이 기준으로 보기`를 눌러야 적용됩니다.
- 추천 이유는 `추천 이유 ?` 말풍선으로 확인합니다.

## 최신 시연 순서

1. Wallet 첫 화면에서 `Starbucks에서 결제하시나요?`와 Top-1 추천을 확인합니다.
2. `추천 보기`를 눌러 추천 상세 하단 시트를 엽니다.
3. `다른 기준 보기`에서 추천 기준을 바꾸고, 카드/혜택 변화 미리보기를 확인한 뒤 `이 기준으로 보기`를 누릅니다.
4. `다른 매장이에요`를 눌러 CU, Olive Young, 백화점 후보 중 하나를 고르고 `이 매장으로 보기`로 적용합니다.
5. 혜택 조합 결제를 선택해 적립/NFC 결제 흐름을 진행합니다.
6. 결제 후 결과 화면에서 예상 혜택, 실적 금액, 카드사 앱 확인 CTA를 확인합니다.

## 시나리오 매핑

| 데모 기준 | 의도 | 코드 키 |
| --- | --- | --- |
| 혜택 | 포인트, 즉시할인, 남은 혜택을 묶어 가장 직관적인 절감 효과를 보여줍니다. | `max_benefit` |
| 마일리지 | 현금 할인보다 항공 마일리지 적립을 우선합니다. | `mileage` |
| 실적 | 다음 달 혜택 조건을 채우는 데 도움이 되는 카드를 우선합니다. | `performance_fill` |

## 현재 결정 로그

- 카드 실명은 발표 이해도를 위해 현재 데모에서는 유지합니다.
- 추가 카드 이미지는 이번 브랜치 범위에서는 넣지 않고, 이미지가 없는 카드는 fallback 카드 디자인을 사용합니다.
- 최신 Samsung Wallet 캡처는 필수 입력으로 두지 않고, 현재 S26 비율과 One UI식 하단 시트 패턴을 기준으로 검수합니다.
- 페이 플래너 별도 mock 화면은 만들지 않고, 카드사 앱 확인 CTA로 정리합니다.
- 회귀 검수는 `npm run qa`로 실행합니다.
