# SWAP Samsung Wallet Demo

Samsung Wallet 안에서 결제 직전의 카드, 쿠폰, 멤버십, 실적 상태를 정리해 보여주는 SWAP 데모입니다.

DEMO: [https://rookie-swap.netlify.app/](https://rookie-swap.netlify.app/)

## 최근 반영 내용

- GS칼텍스 주유소 메인 시나리오 정리
- 삼성 iD STATION 10% 할인 가능, 잔여 한도, GS&POINT 적립 안내
- 결제 전에는 실제 결제금액을 가정하지 않고, 결제 후 승인금액 기준으로 혜택 계산
- 뱅크샐러드 협업 전제의 `powered by 뱅크샐러드` 표기 추가
- 혜택 조합 결제와 직접 결제의 쿠폰/멤버십 단계 UX 통일
- 결과 화면을 결제금액/적립/카드 혜택 중심으로 정리
- 신한 Deep Oil처럼 사용자가 정유사를 직접 선택하는 카드의 `정유사 선택 확인` 케이스 추가
- `#swapAssist` 접힘 상태 레이아웃 수정
- 캡쳐 산출물 정리: `demo_captures/`, `demo_video/`는 폴더만 유지

## 실행 및 검수

```bash
npm install
npm run qa
```

`npm run qa`는 Puppeteer로 주요 결제/추천/결과 흐름을 검증합니다.

## 최신 시연 순서

1. Wallet 첫 화면에서 `GS칼텍스 삼평주유소에서 결제하시나요?`와 추천 CTA를 확인합니다.
2. `추천 보기`를 눌러 추천 상세 하단 시트를 엽니다.
3. `다른 기준 보기`에서 `혜택`, `마일리지`, `실적` 기준을 전환하고 미리보기를 확인합니다.
4. `다른 매장이에요`를 눌러 인접 매장 후보를 고르고 `이 매장으로 보기`로 적용합니다.
5. 혜택 조합 결제를 선택해 쿠폰/적립/NFC 결제 순서를 진행합니다.
6. 직접 카드 결제에서 `쿠폰/멤버십 사용하기`를 선택하면 혜택 결제와 같은 단계형 플로우로 전환되는지 확인합니다.
7. 결제 후 결과 화면에서 결제금액, GS&POINT 적립, 카드 혜택, 실적 진행률, 뱅크샐러드 확인 CTA를 확인합니다.

## 추천 기준 매핑

| 데모 기준 | 의도 | 코드 키 |
| --- | --- | --- |
| 혜택 | 현재 매장/카드 상태에서 가장 실용적인 혜택 조합 | `max_benefit` |
| 마일리지 | 현금 할인보다 마일리지 적립을 우선 | `mileage` |
| 실적 | 아직 미충족이고 목표까지 가까운 카드 우선 | `performance_fill` |

## 데이터/논리 원칙

- 결제 전에는 실제 승인금액을 모르므로 원 단위 절감액을 가정하지 않습니다.
- 결제 전에는 할인율, 잔여 한도, 실적 충족 여부, 보유 포인트, 쿠폰 조건을 보여줍니다.
- 결제 후에는 실제 승인금액 기준으로 할인액/포인트 적립을 계산해 보여줍니다.
- 카드사 청구할인, 실적 산입, 최종 적용 금액은 카드사 기준에 따라 달라질 수 있으므로 뱅크샐러드 앱 확인 CTA를 둡니다.
- 사용자가 직접 혜택 대상을 선택하는 카드, 예를 들어 신한 Deep Oil, 은 현재 매장과 카드사 선택 정유사가 맞는지 확인하는 문구로 처리합니다.
