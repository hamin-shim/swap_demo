# SWAP 추천 로직 시나리오

작성일: 2026-07-12

이 문서는 현재 데모에 등록된 카드 4종을 기준으로, 결제 전 추천과 결제 후 결과 계산을 분리해서 설명하기 위한 발표용 시나리오입니다.

## 핵심 전제

삼성월렛은 결제 전에 사용자가 얼마를 결제할지 알 수 없습니다. 따라서 결제 전 화면에서 특정 원 단위 절감액을 먼저 말하면 안 됩니다.

결제 전에는 뱅크샐러드 콜라보를 통해 확인 가능한 사용자의 카드 상태만 사용합니다.

- 전월 실적 충족 여부
- 이번 달 할인 한도 사용액
- 남은 할인 한도
- 보유 쿠폰
- 보유 포인트
- 멤버십/적립 가능 여부

원 단위 절감액은 결제 후 실제 승인 금액이 생긴 뒤에 계산합니다.

## 보유 카드 기준

| 카드 | 결제 전 판단 기준 | 데모상 현재 상태 |
| --- | --- | --- |
| KB국민 주유패스 | 리터당 할인, 잔여 월 한도 | 리터당 100원 할인 가능. 월 한도 15,000원 중 12,200원 사용, 잔여 2,800원 |
| 삼성 iD STATION | GS칼텍스 제휴 할인율, 실적 충족, 잔여 월 한도, GS&POINT 적립 | 전월 실적 충족. 월 할인 한도 30,000원 중 2,500원 사용, 잔여 27,500원. GS&POINT는 데모 기준 리터당 2P 적립 |
| 신한 Deep Oil | 선택 정유사 조건 | 주유 특화 카드지만 현재 선택 정유사 조건 확인 필요 |
| 현대카드 M BOOST | M포인트 보유량, 사용 가능률 | M포인트 18,400P 보유. 배스킨라빈스에서 결제금액의 50%까지 사용 가능 |

## 시나리오 1. GS칼텍스 삼평주유소

### 결제 전 현재 상태 규명

- 현재 위치는 `GS칼텍스 삼평주유소`입니다.
- 결제 전이므로 실제 주유금액은 아직 모릅니다.
- 뱅크샐러드 기준으로 삼성 iD STATION은 전월 실적을 충족했고, GS칼텍스 할인 잔여 한도가 `27,500원` 남아 있습니다.
- 따라서 결제 전 추천 문구는 `4,500원 할인 예상`이 아니라 `10% 할인 가능`이어야 합니다.

### 카드별 판단

| 카드 | 결제 전 표시 | 판단 |
| --- | --- | --- |
| 삼성 iD STATION | `10% 할인 가능` | GS칼텍스 10% 결제일 할인 가능. 잔여 한도 27,500원까지 적용 가능해서 후보 중 가장 강함 |
| KB국민 주유패스 | `리터당 100원` | 주유 할인은 가능하지만 잔여 한도가 2,800원뿐이라 할인 폭이 제한됨 |
| 신한 Deep Oil | `정유사 확인` | 선택 정유사가 GS칼텍스인지 확인되어야 확정 가능 |
| 현대카드 M BOOST | `기본 2% 적립` | 직접 할인보다 기본 포인트 적립 중심 |

### 결제 후 계산

실제 결제가 끝나면 승인 금액을 기준으로 원 단위 절감액을 계산합니다.

데모 결과 예시:

- 실제 주유금액: `45,000원`
- 삼성 iD STATION 할인율: `10%`
- 잔여 할인 한도: `27,500원`
- 실제 절감액: `45,000원 x 10% = 4,500원`
- GS&POINT 적립: `45,000원 ÷ 1,600원/L ≈ 28L`, `28L x 2P = 56P`

따라서 결제 후 결과 화면에서만 `카드할인 4,500원 + GS&POINT 56P`처럼 말할 수 있습니다.

발표 문장 예시:

> 결제 전에는 주유금액을 모르기 때문에 SWAP은 원 단위 할인액을 찍지 않습니다. 대신 뱅크샐러드에서 확인한 카드 상태를 바탕으로, 삼성 iD STATION이 GS칼텍스 10% 할인 대상이고 잔여 한도가 27,500원 남았다는 점을 보여줍니다. 결제 후 실제 승인 금액이 45,000원으로 확인되면 그때 10%를 환산해 4,500원을 아꼈다고 정리합니다.

## 시나리오 2. 배스킨라빈스 판교역점

### 결제 전 현재 상태 규명

- 현재 위치는 `배스킨라빈스 판교역점`입니다.
- 결제 전이므로 실제 주문 금액은 아직 모릅니다.
- 뱅크샐러드 기준으로 현대카드 M포인트 `18,400P`를 보유 중입니다.
- 현대카드 M BOOST는 배스킨라빈스에서 결제금액의 `50%`까지 M포인트 사용을 요청할 수 있습니다.
- 따라서 결제 전 추천 문구는 `5,000원 할인 예상`이 아니라 `M포인트 50% 사용 가능`이어야 합니다.

### 카드별 판단

| 카드 | 결제 전 표시 | 판단 |
| --- | --- | --- |
| 현대카드 M BOOST | `M포인트 50% 사용 가능` | 보유 M포인트 18,400P가 충분하고, 결제금액의 절반까지 사용 가능 |
| 삼성 iD STATION | `2,000원 쿠폰 가능` | 삼성카드 LINK 쿠폰이 있으면 18,500원 이상 결제 시 2,000원 할인 가능 |
| KB국민 주유패스 | `기본 적립 가능` | 배스킨라빈스 직접 할인 근거가 약함 |
| 신한 Deep Oil | `업종 혜택 확인` | 주유소 혜택 중심이라 이번 업종과 맞지 않음 |

### 결제 후 계산

실제 결제가 끝나면 승인 금액을 기준으로 M포인트 사용액을 계산합니다.

데모 결과 예시:

- 실제 결제금액: `10,000원`
- M포인트 사용률: `50%`
- 보유 M포인트: `18,400P`
- 실제 사용액: `10,000원 x 50% = 5,000P`

따라서 결제 후 결과 화면에서만 `방금 결제로 5,000원 아꼈어요`라고 말할 수 있습니다.

발표 문장 예시:

> 배스킨라빈스에서도 결제 전에는 주문 금액을 모르기 때문에 5,000원을 먼저 말하지 않습니다. 대신 보유 M포인트가 18,400P이고, 이 매장에서 결제금액의 50%까지 사용할 수 있다는 조건을 보여줍니다. 결제 후 실제 승인 금액이 10,000원으로 확인되면 그때 50%를 환산해 M포인트 5,000P를 사용했다고 정리합니다.

## UI 원칙

| 시점 | 보여줄 정보 | 보여주면 안 되는 정보 |
| --- | --- | --- |
| 결제 전 | 할인율, 사용 가능률, 잔여 한도, 보유 포인트, 쿠폰 조건 | 실제 결제금액을 가정한 원 단위 예상 절감액 |
| 결제 후 | 실제 승인 금액 기준 절감액, 사용 포인트, 적용된 쿠폰/적립 | 결제 전 추정처럼 보이는 모호한 예상액 |

정리하면 SWAP의 추천 논리는 `현재 카드 상태로 어떤 혜택을 받을 수 있는가`를 먼저 보여주고, `실제로 얼마를 아꼈는가`는 결제 후에 계산하는 구조입니다.

## 마일리지와 실적 기준

마일리지 기준도 같은 원칙을 따릅니다. 결제 전에는 `45마일 적립 예상`처럼 숫자를 찍지 않습니다. 대신 `마일리지 적립 가능`, `결제 후 승인금액과 카드사 적립률 기준으로 확정`처럼 조건만 보여줍니다.

실적 기준은 부족액과 달성률을 중심으로 보여줍니다. 예를 들어 삼성 iD STATION은 `388,000원/400,000원`, 즉 다음 주유 혜택 조건까지 `12,000원`만 남았기 때문에 실적 기준에서 좋은 후보가 됩니다. 이때 문구는 `실적이 부족합니다`보다 `다음 주유 혜택까지 12,000원 남았어요`처럼 부드럽게 표현합니다.
