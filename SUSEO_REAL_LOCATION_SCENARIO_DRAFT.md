# Suseo Real Location Scenario Draft

작성일: 2026-07-13  
상태: 수서 SK 시나리오 구현 브랜치 기준 초안. 결제 전 화면에는 승인금액을 노출하지 않고, 결제 후 결과 계산에만 내부 승인금액을 사용한다.

## 1. 문서 배경

이 문서는 기존 SWAP 데모의 가상/판교 주유소 맥락을 실제 수서 위치 기반 시나리오로 바꾸기 위해 만들었다. 비개발 팀원이 봐도 `어느 위치에서`, `어떤 카드가 왜 추천되고`, `사용자는 어떤 버튼을 눌러야 하며`, `결제 후 무엇을 확인해야 하는지`를 빠르게 이해하는 것이 목적이다.

현재 구현 브랜치의 기준 장소:

- SK엔크린 중앙에너비스 수서지점
- 배스킨라빈스 강남수서점
- 투썸플레이스 강남수서점

핵심 원칙은 하나다. 결제 전에는 사용자가 얼마를 결제할지 모르므로 원 단위 혜택을 노출하지 않고, 결제 후 승인금액이 생긴 뒤에만 할인/포인트를 계산한다.

### 위치 해석

사용자는 실제로 SK 주유소에 있다고 본다. 다만 근처에 배스킨라빈스가 매우 가까이 있고, 투썸플레이스도 후보로 잡힐 수 있다.

따라서 데모의 위치 인식 UX는 다음처럼 설계하는 것이 현실적이다.

```text
현재 위치 후보: SK엔크린 중앙에너비스 수서지점
근접 후보 1: 배스킨라빈스 강남수서점
근접 후보 2: 투썸플레이스 강남수서점
```

첫 화면 문구 후보:

- `SK엔크린 중앙에너비스 수서지점에서 결제하시나요?`
- `근처 매장이 여러 곳 있어요. SK엔크린으로 볼까요?`
- `SK 주유소로 인식했어요`

현재 데모 톤을 유지한다면 1번이 가장 직접적이다.

### SK 주유 쿠폰 / OK캐쉬백 현실성 검증

결론부터 말하면 첫 구현 조합은 현실성이 있다. 다만 화면에서 표현할 때는 `자동으로 모든 정보가 연동됨`처럼 보이게 하기보다, `보유 쿠폰과 멤버십을 결제 전에 같이 꺼내준다`는 톤이 더 안전하다.

| 항목 | 확인 내용 | 데모 표현 |
| --- | --- | --- |
| OK캐쉬백 | 삼성페이/삼성월렛에 미지원 멤버십도 직접 등록하면 바코드 생성 후 적립/사용 가능. 단, 직접 등록 멤버십은 현재 포인트나 이용내역 조회가 제한될 수 있음 | `OK캐쉬백 바코드 제시` 또는 `OK캐쉬백 적립 준비` |
| SK 주유쿠폰 | SK 모바일 주유권/주유쿠폰 상품은 존재한다. 전국 SK주유소/충전소 사용 상품도 있으나, 쿠폰 종류별로 사용 가능 주유소와 조건이 달라질 수 있음 | `SK 주유쿠폰 2,000원 사용 가능` 대신 `보유한 SK 주유쿠폰을 같이 확인해요`처럼 조건형으로 표시 |
| 주유소 현장 플로우 | 셀프 주유기에서 모바일 바코드/쿠폰번호를 스캔하거나 입력하는 흐름이 가능하다. 주유소/쿠폰별 UI 차이는 있을 수 있음 | `쿠폰/멤버십 먼저` 단계에서 쿠폰과 OK캐쉬백을 카드 위에 함께 노출 |
| 포인트 적립률 | SK에너지의 머핀/엔크린 포인트는 OK캐쉬백으로 전환/통합되었다. 리터당 적립률은 카드/멤버십/정책에 따라 달라질 수 있으므로, 데모에서는 결제 전 금액 추정 없이 `결제 후 리터/승인금액 기준으로 계산`하는 것이 안전함 | 결제 전: `OK캐쉬백 적립 가능`; 결제 후: `실제 주유량 기준 OK캐쉬백 00P 적립` |

따라서 첫 구현 추천은 아래처럼 확정한다.

```text
현재 위치: SK엔크린 중앙에너비스 수서지점
기본 기준: 혜택
추천 카드: 신한 Deep Oil
카드 조건: 선택 정유사 SK에너지, 전월 실적 충족, 주유 할인 한도 남음
멤버십: OK캐쉬백 바코드 적립
쿠폰: 보유한 SK 주유쿠폰 확인
결제 전 문구: SK에너지 10% 할인 가능
쿠폰/멤버십 단계: SK 주유쿠폰 -> OK캐쉬백 -> 결제
결제 후 문구: 실제 승인금액 기준 청구할인 + 실제 주유량 기준 OK캐쉬백 적립
```

주의할 점:

- 삼성월렛이 OK캐쉬백의 잔액/이용내역까지 직접 조회하는 화면처럼 만들면 부정확하다.
- 뱅크샐러드 협업으로 알 수 있는 것은 카드 실적/한도/결제내역 기반 혜택 계산이다. OK캐쉬백 잔액이나 SK 쿠폰 보유 여부는 사용자 등록/쿠폰함/멤버십 연동 가정이 따로 필요하다.
- SK 주유쿠폰은 "원래 존재하는 상품"으로 두되, 특정 사용 가능 여부는 쿠폰 조건 확인 후라는 뉘앙스를 남기는 편이 현실적이다.

## 8. 심사위원 예상 질문과 방어 시나리오

이 데모는 "삼성월렛이 모든 금융/멤버십 정보를 직접 가진다"가 아니라, `Samsung Wallet 결제 맥락 + 뱅크샐러드 마이데이터/카드 혜택 계산 + 사용자가 등록한 쿠폰/멤버십 바코드`를 결제 직전에 한 화면으로 묶어주는 제안이다.

### 가장 세게 들어올 질문

| 예상 질문 | 비판 포인트 | 방어 논리 | 데모에서 더 보여줄 것 |
| --- | --- | --- | --- |
| 삼성전자가 카드 실적/한도/혜택을 어떻게 알아요? | 삼성월렛 단독으로는 카드사별 실적/한도 정보를 알기 어렵다 | 그래서 `powered by 뱅크샐러드`가 필요하다. 금융 마이데이터는 카드 결제내역, 청구금액, 포인트 현황, 카드별 승인 상세정보 같은 데이터를 다룰 수 있다 | 추천 카드 하단에 `뱅크샐러드가 카드 실적/한도 기준으로 계산했어요` 툴팁 |
| Deep Oil이 SK에너지 선택인지 어떻게 알아요? | 사용자가 카드사 앱에서 고른 선택 정유사 정보는 민감하고, 마이데이터 기본 API만으로 항상 확정하기 어렵다 | 데모에서는 사용자가 뱅크샐러드/카드사 연동으로 `선택 정유사: SK에너지`를 등록 또는 확인했다는 전제를 둔다. 확실하지 않을 때는 "확인 필요" 상태로 낮춘다 | `선택 정유사 SK에너지 확인됨` 또는 `정유사 선택 확인 필요`의 두 가지 상태 |
| 결제 전에는 금액을 모르는데 왜 할인액을 말해요? | 이전 데모의 가장 큰 허점. 결제 전 금액 추산은 위험하다 | 결제 전에는 `10% 할인 가능`, `주유 할인 대상 이용금액 한도 150,000원 중 92,000원 남음`, `쿠폰 보유`처럼 조건만 말한다. 결제 후 승인금액이 생긴 뒤에만 `4,500원 할인 예정`처럼 금액 환산한다 | 결제 전/후 문구를 명확히 분리 |
| OK캐쉬백까지 자동으로 적립되나요? | 월렛이 OK캐쉬백 잔액/적립내역까지 자동 조회하는 것처럼 보이면 과장이다 | 삼성월렛에 직접 등록한 멤버십 바코드로 적립/사용은 가능하지만, 포인트 잔액/이용내역 조회는 제한될 수 있다. 따라서 "자동 계산"보다 "바코드 제시"가 맞다 | 결제 단계에서 `OK캐쉬백 바코드 제시`로 표시하고, 사후 화면은 `주유소/OK캐쉬백 기준 적립 확인 필요` 톤 |
| SK 주유쿠폰이 정말 이 주유소에서 되나요? | 모바일 주유권은 상품/주유소/쿠폰 조건마다 제한이 있다 | SK 모바일 주유권은 존재하지만, 특정 쿠폰은 사용 가능 매장과 조건 확인이 필요하다. 그래서 데모에서는 `보유 쿠폰 확인` 또는 `사용 가능하면 먼저 제시`로 둔다 | 쿠폰 카드에 `사용 전 주유소 조건 확인` 작은 문구 |
| 위치가 배스킨라빈스랑 가까우면 오인식할 수 있지 않나요? | 실제 위치 기반 추천은 오탐 가능성이 크다 | 그래서 첫 화면에서 매장명을 크게 보여주고 사용자가 `다른 매장이에요`를 누를 수 있게 한다. 위치 자동 확정이 아니라 "현재 후보 확인" 구조다 | `SK엔크린 중앙에너비스 수서지점에서 결제하시나요?` + `다른 매장이에요` |
| 카드 혜택 추천이 틀리면 책임은 누가 지나요? | 카드사 정책, 전표 매입, 제외 업종, 한도 반영 타이밍 때문에 오차 가능 | 결제 전은 `예상/가능`, 결제 후도 `청구할인 예정`으로 표시하고, 최종 확인은 뱅크샐러드/카드사 앱으로 안내한다 | 결과 화면 하단의 회색 안내: `카드사 확정 금액은 뱅크샐러드 앱에서 더 정확히 확인할 수 있어요` |

### 추가로 보여주면 좋은 방어 시나리오

#### A. 정상 추천 시나리오

```text
상황: 사용자가 SK엔크린 중앙에너비스 수서지점에 있음
사용자 상태: Deep Oil 선택 정유사 SK에너지, 전월 실적 충족, 주유 할인 한도 남음
결제 전: "SK에너지 10% 할인 가능"
쿠폰/멤버십: SK 주유쿠폰 확인 -> OK캐쉬백 바코드 제시 -> Deep Oil 결제
결제 후: 실제 승인금액 기준 청구할인 예정 금액만 계산
```

이 시나리오는 첫 구현의 메인 플로우다.

#### B. 선택 정유사 불일치 시나리오

```text
상황: 현재 위치는 SK 주유소
사용자 상태: Deep Oil 선택 정유사가 SK에너지가 아니거나 미확인
결제 전: "이 카드는 선택 정유사 확인이 필요해요"
추천 전환: KB국민 주유패스 또는 삼성 iD STATION 등 선택 정유사 의존도가 낮은 후보 노출
```

이 시나리오를 넣으면 "왜 현재 위치와 카드 설정을 같이 봐야 하는지"가 분명해진다.

#### C. 할인 한도 소진 시나리오

```text
상황: Deep Oil 조건은 맞지만 이번 달 주유 할인 한도를 거의 다 씀
결제 전: "10% 할인 조건은 맞지만 이번 달 남은 할인 가능액이 800원이에요"
추천 전환: 포인트형 또는 실적 충족형 카드
```

이 시나리오는 뱅크샐러드 협업의 필요성을 가장 잘 보여준다. 단순 카드 혜택표만 보면 Deep Oil이 좋아 보이지만, 개인의 현재 한도 상태까지 보면 추천이 바뀔 수 있다.

#### D. 쿠폰 조건 미확정 시나리오

```text
상황: 사용자가 SK 주유쿠폰을 보유했지만 사용 가능 주유소/최소 결제 조건이 미확정
결제 전: "보유한 SK 주유쿠폰을 확인해볼게요"
쿠폰 단계: "이 쿠폰은 사용 전 주유기/직원 확인이 필요해요"
결제 후: 쿠폰 적용 여부를 승인금액 기준으로 반영
```

이 시나리오는 "쿠폰 자동 적용" 오해를 막는다.

#### E. 위치 오인식 시나리오

```text
상황: SK 주유소와 배스킨라빈스가 가까워 위치 후보가 겹침
첫 화면: "SK엔크린 중앙에너비스 수서지점에서 결제하시나요?"
사용자 선택: "다른 매장이에요"
대체 후보: 배스킨라빈스 강남수서점, 투썸플레이스 강남수서점
추천 전환: 해피포인트/M포인트/카페 혜택 중심
```

이 시나리오는 위치 기반 서비스의 현실적인 불확실성을 숨기지 않고, 사용자의 한 번 확인으로 해결한다.

### 심사 답변용 한 문장

```text
SWAP은 결제 전 금액을 예측하는 서비스가 아니라, 지금 위치와 사용자의 카드 상태를 바탕으로 "어떤 카드/쿠폰/멤버십을 먼저 꺼내야 하는지"를 알려주고, 결제 후 실제 승인금액 기준으로 혜택을 정리하는 결제 보조 레이어입니다.
```

### 구현 우선순위 제안

1. 메인 플로우는 `정상 추천 시나리오` 하나만 먼저 구현한다.
2. 추천 이유 툴팁에 `뱅크샐러드가 계산한 정보`와 `결제 전에는 금액을 추정하지 않음`을 넣는다.
3. 카드 추천 카드 하단에 `조건이 바뀌면 추천도 바뀔 수 있어요`를 작은 회색 문구로 둔다.
4. 이후 2차 구현에서 `선택 정유사 불일치`, `한도 소진`, `위치 오인식`을 데모 분기 또는 QA용 상태로 추가한다.

## 9. 내부 승인금액 시나리오

아래 금액은 데모가 결제 후 결과 화면을 계산하기 위한 내부 truth table이다. 결제 전 추천 화면, 추천 이유, 기준 변경 화면에서는 절대 원 단위 예상 결제액으로 노출하지 않는다.

| 위치 | 내부 결제 상황 | 결제 전 노출 | 결제 후 계산 |
| --- | --- | --- | --- |
| SK엔크린 중앙에너비스 수서지점 | 휘발유 45,000원 주유, 약 23L로 환산 | `SK에너지 10% 할인 가능`, `주유 할인 대상 이용금액 한도 150,000원 중 92,000원 남음`, `SK 주유쿠폰 확인`, `OK캐쉬백 바코드 제시` | 카드할인 4,500원 + OK캐쉬백 46P |
| 배스킨라빈스 강남수서점 | 아이스크림/디저트 10,000원 결제 | `M포인트 50% 사용 가능`, `M포인트 18,400P 보유`, `해피포인트 적립` | M포인트 5,000P 사용 + 해피포인트 적립 |
| 투썸플레이스 강남수서점 | 커피/디저트 12,800원 결제 | `포인트 적립 가능`, `투썸하트 적립`, Deep Oil 커피 혜택은 직접 대상 아님 | M포인트 128P + 투썸하트 적립 |

계산 원칙:

- 결제 전에는 할인율, 잔여 한도, 쿠폰/멤버십 존재, 선택 정유사 일치 여부만 보여준다.
- 결제 후에는 실제 승인금액이 생긴 뒤에만 원/P 단위 금액을 표시한다.
- OK캐쉬백 46P는 데모 계산 편의를 위해 `45,000원 주유 ~= 23L`, `리터당 2P`로 둔다. 실제 적립률과 한도는 정책에 따라 달라질 수 있으므로 결과 화면 도움말에 이 한계를 남긴다.
- 투썸은 신한 Deep Oil 커피 생활서비스의 직접 대상이 아니므로, 카페 후보에서는 포인트형 카드와 투썸하트 적립 중심으로 보여준다.

## 10. 이미지 요청 후보

우선 이미지가 필요한 카드 4개:

1. 신한카드 Deep Oil
2. KB국민 주유패스
3. 삼성 iD STATION
4. 현대카드 M BOOST

추가로 가능하면:

- OK캐쉬백 아이콘
- 해피포인트 아이콘
- 투썸하트/투썸 멤버십 아이콘

## 11. 확인 필요

- 신한 Deep Oil의 선택 정유사를 SK에너지로 설정한 사용자로 둘지, 선택 정유사 확인이 필요한 사용자로 둘지 결정.
- 삼성 iD STATION을 어떤 생활/F&B 후보로 둘지, 또는 주유소에서는 약한 후보로만 남길지 결정.
- 배스킨라빈스 시나리오에서 현대카드 M BOOST의 M포인트 사용을 메인으로 유지할지, 삼성 iD STATION 쿠폰 후보를 더 강조할지 결정.
