SCREEN 01 Wallet Home

메인 화면: 추천은 결제 버튼을 대체하지 않고 위에 얹힙니다.

1

추천 보기

현재 매장과 보유 카드 기준의 추천 조합을 바텀시트로 엽니다.

2

다른 매장이에요

위치 후보가 틀릴 수 있음을 인정하고 사용자가 실제 결제 매장을 선택하게 합니다.

3

이 카드로 결제

추천을 쓰지 않고 기존 Samsung Pay 결제 흐름으로 바로 진입할 수 있습니다.

방어 논리: SWAP은 자동결제나 광고 CTA가 아니라, 사용자가 확인할 수 있는 선택 보조 레이어입니다.
SCREEN 02 Recommendation sheet

추천 보기: 카드·쿠폰·멤버십을 한 조합으로 묶습니다.

1

카드

신한 Deep Oil은 선택 정유사 SK에너지, 전월 실적 충족, 주유 할인 대상 이용금액 잔여분이 있어 기본 추천 카드입니다.

2

쿠폰·멤버십

SK 주유쿠폰 확인과 OK캐쉬백 바코드 제시를 같은 결제 흐름 안에 둡니다.

3

이 조합으로 결제

사용자가 명시적으로 선택한 뒤에만 결제 순서 화면으로 넘어갑니다.

방어 논리: 추천 결과는 강제 적용이 아니라 “결제 전 확인 가능한 조합”입니다.
SCREEN 03 Preview before apply

기준과 위치가 바뀌어도, 사용자가 적용할 때만 추천이 바뀝니다.

다른 기준 보기

SCREEN 04 Payment sequence

결제 실행: 쿠폰 → 멤버십 → Samsung Pay 순서로 안내합니다.

1. 쿠폰 먼저

직원에게 먼저 제시해야 하는 혜택 바코드부터 보여줍니다.

2. 멤버십 적립

OK캐쉬백처럼 별도 앱을 열기 쉬운 항목을 같은 흐름에 둡니다.

3. NFC 결제

마지막은 기존 Samsung Pay 인증과 NFC 결제 구조를 유지합니다.

SCREEN 05 Payment result

결제 후: 승인금액 기준으로 예상 혜택과 실적 상태를 정리합니다.

1

실제 승인금액 기준

45,000원 승인 후에만 10% 할인과 OK캐쉬백 적립을 금액으로 계산합니다.

2

실적 상태

이번 달 실적을 금액 기준으로 보여주고, 다음 추천에 참고합니다.

3

카드사 앱 확인

청구할인, 실적 산입, 잔여 한도는 카드사 기준에 따라 최종 확인할 수 있게 연결합니다.

방어 논리: 혜택 보장이 아니라 예상 정리와 사후 확인 경로 제공입니다.
DESIGN RATIONALE 01 Scenario realism

보유 카드 4종은 평균이 아니라 추천 로직 검증 세트입니다.

DESIGN RATIONALE 02 Pre vs post payment

결제 전에는 가능 조건, 결제 후에만 원 단위 금액을 말합니다.

DESIGN RATIONALE 03 User control

추천이 광고처럼 느껴지지 않도록, 사용자가 노출 강도를 조절합니다.

DESIGN RATIONALE 04 Fallback and verification

모르면 단정하지 않고, 수정·확인·사후 검증 루프를 제공합니다.

DATA BOUNDARY What SWAP can safely claim

데이터는 “확인 가능 / 확인 필요 / 사후 계산”으로 나눠야 현실적입니다.

A 확인 가능

  • 등록 카드와 카드 이용내역
  • 전월 실적 충족 여부
  • 이번 달 실적 진행률
  • 포인트 잔액과 일부 잔여 한도
  • Wallet에 등록된 멤버십/쿠폰

B 확인 필요

  • 선택 정유사처럼 카드사 앱 설정이 필요한 조건
  • 쿠폰 최소 결제액과 사용 가능 매장
  • 금액권 쿠폰 사용 후 잔액
  • 청구할인 최종 반영 여부
  • 실적 산입 제외 업종 여부

C 사후 계산

  • 실제 승인금액 기준 할인액
  • 주유량 환산 포인트
  • 결제 후 실적 진행률
  • 추천 수락 여부와 다음 추천 반영
  • 카드사 앱 최종 확인 연결

핵심은 “모든 것을 안다”가 아니라, 확정성 수준에 맞는 UI 언어를 쓰는 것입니다.

MVP SCOPE Practical rollout

MVP는 완전 자동화가 아니라, 판단 비용이 큰 순간부터 줄입니다.

1

1차 MVP

등록 카드, 공개 혜택 조건, 위치 업종 후보, 일부 실적/한도 정보를 바탕으로 추천 확인, 기준 변경, 매장 수정 UX를 구현합니다.

2

제휴 확장

카드사, 멤버십, 쿠폰 제휴를 늘려 사용 확정, 금액권 잔액, 청구할인 확인 정확도를 높입니다.

3

개인화 고도화

사용자의 혜택 선호, 결제 습관, 위치 맥락을 온디바이스 기반으로 학습해 추천 기준을 더 자연스럽게 제안합니다.

발표에서는 “모든 혜택 자동화”보다 “Wallet을 여는 순간의 결제 판단 비용을 먼저 줄인다”로 말하는 편이 설득력 있습니다.

EXPECTED QUESTIONS Executive review checklist

심사위원이 물을 가능성이 높은 질문과 답변 방향

보유 카드 4장은 평균 사용자를 대표하나요?

평균이 아니라 혜택 민감형 사용자의 핵심 판단 케이스를 검증하는 카드 세트입니다.

결제 전인데 금액 혜택을 어떻게 알 수 있나요?

전에는 조건만 보여주고, 실제 금액은 승인 후 결과 화면에서만 계산합니다.

Samsung Pay에 광고처럼 보이지 않나요?

추천은 접거나 끌 수 있고, 기존 카드 바로 결제 경로를 유지합니다.

위치와 금융 데이터 결합이 민감하지 않나요?

상시 추적이 아니라 결제 맥락의 후보 인식이며, 사용자가 매장을 수정할 수 있습니다.

쿠폰/멤버십 사용 확정은 어떻게 압니까?

바코드 제시와 사용 확정을 분리하고, 제휴 데이터가 없으면 확인 중 또는 앱 확인으로 처리합니다.

이건 카드사나 핀테크가 할 일 아닌가요?

핀테크도 계산은 가능하지만, Wallet을 여는 결제 직전 순간에는 삼성의 진입점이 더 자연스럽습니다.
