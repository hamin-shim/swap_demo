3. AI와 추천 로직의 실체 Q12 Answer

결제 전에는 “얼마 아꼈다”가 아니라 “조건이 맞다”를 보여줍니다

Before Pay 결제 전 표시

  • 매장/업종 후보
  • 할인율, 사용 가능률, 잔여 한도
  • 보유 포인트와 쿠폰 조건
  • 실적 목표까지 남은 금액

좋은 문구

“SK에너지 10% 할인 가능”
“주유 할인 한도 92,000원 남음”

피할 문구

“4,500원 할인 확정”
“가장 좋은 카드입니다”

After Pay 결제 후 표시

  • Samsung Pay 승인금액
  • 승인금액 기준 예상 할인액
  • 포인트 사용/적립 예상
  • 카드사 앱 최종 확인 CTA

좋은 문구

“승인금액 45,000원 기준 청구할인 예정”

신뢰 장치

“최종 적용은 카드사 기준으로 확인할 수 있어요”

3. AI와 추천 로직의 실체 Q13-Q15 Answer

복잡한 카드 조건은 확정하지 않고 상태값과 책임 주체로 나눕니다

조건 처리 상태

확인됨

파트너 데이터 또는 Wallet 보유 자산으로 확인 가능한 조건입니다.

가능

공개 약관/보유 정보상 적용 가능성이 높지만 결제 후 확정됩니다.

확인 필요

선택 정유사, MCC 제외, 카드사 설정처럼 최종 조건 확인이 필요합니다.

제외

업종 불일치, 한도 소진, 포인트 부족처럼 추천에서 제외할 조건입니다.

책임 주체

  • 파트너: 금융 데이터, 혜택 규칙, 추천 콘텐츠 생성
  • 삼성: 동의, context 전송, Wallet 표시/수정/결제 UX
  • 카드사/멤버십사: 최종 승인, 청구할인, 적립 확정
  • 화면: `예상`, `가능`, `확인 필요` 톤으로 책임 경계 표시
6. UX와 사용성 신뢰 Q25-Q27 Answer

신뢰 UX는 “선택권, 미리보기, 최종 확인” 세 단계로 만듭니다

1. 추천은 접거나 무시 가능

기존 카드 결제 버튼을 유지해 추천이 결제를 가로막지 않음.

2. 기준 변경은 미리보기 후 적용

기준을 누르는 즉시 카드가 바뀌지 않고 사용자가 적용해야 반영.

3. 결과는 카드사 확인으로 닫음

청구할인/실적 확정은 카드사 기준임을 노출.

8. 화면 기반 질문 답변 Q34-Q35 Answer

매장 인식은 자동 확정이 아니라 후보 제안과 사용자 확인 구조입니다

High

신뢰도 높음

대표 매장명을 제안하되 `에서 결제하시나요?`처럼 확인형 문구로 표시합니다.

Medium

후보 겹침

근처 매장 후보를 보여주고, 사용자가 선택해야 추천 조합이 바뀝니다.

Low

불확실

브랜드 혜택 대신 범용 카드/업종 추천으로 낮추고 오추천 리스크를 줄입니다.

Learn

사용자 수정 반영

오인식 수정은 다음 후보 정렬의 단말 내 선호 신호로 반영합니다.

8. 화면 기반 질문 답변 Q36-Q37 Answer

결제 단계는 새 절차를 늘리는 것이 아니라 흩어진 행동을 한 줄로 정리합니다

쿠폰/멤버십 먼저

계산대에서 실제로 말하고 보여줘야 하는 순서대로 제시.

NFC 결제는 기존 방식

AI가 결제를 대신하지 않고 기존 Samsung Pay 승인 흐름 유지.

사후 확인은 불신 방지

결제 후 앱 이동은 실패가 아니라 최종 확정 책임 경계 표시.

7. MVP 범위와 실행 계획 Q30-Q32 Answer

MVP는 모든 혜택 자동화가 아니라 반복 결제 업종의 판단 비용 감소입니다

1

1차 MVP

주유소/F&B/편의점 중심. 등록 카드, 공개 혜택, 파트너 실적/한도 일부 연동.

2

제휴 확장

쿠폰 잔액, 멤버십 사용 확정, 카드사/브랜드 API 연동.

3

개인화 고도화

사용자 선택 이력, 오인식 수정, 선호 기준을 단말 중심으로 학습.

확인률

추천을 열어 본 비율. 첫 화면 문구와 신뢰도 측정.

수락률

추천 조합으로 결제한 비율. 로직 유용성 측정.

수정률

매장/기준 변경 비율. 오인식과 복잡도 측정.

끄기율

추천 기능을 끈 비율. 결제 UX 침범 여부 측정.

전환율

추천 후 실제 Pay 결제로 이어진 비율.

시간

결제까지 걸린 시간. 빠른 결제 훼손 여부 측정.

7. MVP 범위와 실행 계획 Q33 Answer

실패 상태는 숨기지 않고 기존 결제 흐름으로 안전하게 fallback합니다

!

파트너 API 지연

추천을 기다리게 하지 않고 기본 카드 결제로 진입합니다. 추천은 “확인 중”으로 낮춥니다.

X

혜택 조건 불확실

선택형 혜택, MCC 제외, 실적 산입 불확실 항목은 `확인 필요`로 표시합니다.

↺

매장 오인식

추천 결과를 강제 적용하지 않고 후보 리스트에서 사용자가 직접 적용합니다.

✓

결제 후 검증

승인금액 기준으로 다시 계산하고 최종 확정은 카드사/파트너 앱으로 연결합니다.
