# SWAP Demo CTA Report

작성일: 2026-07-11  
기준 파일: `script.js`

## CTA 뜻

CTA는 **Call To Action**의 약자입니다.  
서비스 화면에서는 사용자가 다음 행동을 바로 이해하도록 유도하는 버튼 문구, 배지 문구, 링크 문구를 의미합니다.

예: `4,500원 할인 예상`, `조건 확인`, `실적 12,000원 남음`

## 현재 CTA 생성 규칙

| 조건 | CTA 규칙 | 예시 |
| --- | --- | --- |
| 마일리지 기준 | 마일리지 금액/마일이 있으면 그대로 요약 | `항공 마일리지 45마일` |
| 마일리지 기준인데 효율 낮음 | 대체 확인 문구 | `마일리지 확인` |
| 실적 기준 + 삼성 iD STATION | 다음 조건까지 남은 금액 강조 | `실적 12,000원 남음` |
| 실적 기준 + 금액/목표 있음 | 현재 금액/목표 표시 | `318,000원/500,000원` |
| 혜택 확인 필요 | 조건 확인으로 축약 | `조건 확인` |
| 남은 주유 한도 | 남은 한도만 짧게 표시 | `한도 19,000원` |
| 예상 혜택 금액 | 할인 예상 문구로 표시 | `4,500원 할인 예상` |

## 문구 검수 포인트

| 항목 | 현재 상태 | 제안 |
| --- | --- | --- |
| 할인형 CTA | 금액이 바로 보여서 데모 전달력이 좋음 | 유지 |
| 조건형 CTA | `조건 확인`으로 짧고 무난함 | 유지하되 상세 이유는 바텀시트/카드 설명에서 보완 |
| 마일리지 CTA | 추천 카드는 명확함 | 비추천 카드의 `마일리지 확인`은 조금 밋밋하므로 필요 시 `마일리지 낮음`을 그대로 쓰는 것도 가능 |
| 실적 CTA | 일부 카드가 `318,000원/500,000원`처럼 딱딱하게 보임 | `실적 182,000원 남음`, `실적 충족 완료`처럼 액션형으로 통일 권장 |
| 레거시 기준 | `남은 한도`, `조건 없음` 데이터가 코드에 남아 있음 | UI에서 완전히 숨길지, 개발 검수용으로 둘지 결정 필요 |

## 다음 수정 후보

1. 실적 CTA를 모두 `현재금액/목표금액` 대신 `남은 금액` 또는 `충족 완료`로 통일합니다.
2. S-OIL, EV 충전소의 실적/한도 기준 데이터를 별도로 만들거나, 추천 설정에서 해당 기준을 선택했을 때 “현재 위치에서는 혜택 기준으로 확인” 같은 안내를 둡니다.
3. `남은 한도`, `조건 없음` 레거시 기준이 정말 필요 없으면 데이터와 QA에서 정리합니다.
4. 카드별 CTA와 카드별 설명 문구를 같은 톤으로 맞춥니다. 예: `조건 확인` CTA 아래 설명은 “선택 정유사 조건을 확인해야 해요”처럼 이유를 한 문장으로 보강합니다.
