APPENDIX DRAFT

메인 화면 기준으로 기능을 설명할 때는 “버튼 → 반응 → 설계 의도” 순서가 가장 깔끔합니다.

심사자나 면접관이 보고 싶은 것은 화면의 모든 기능 목록이 아니라, 사용자가 계산대 앞에서 어떤 선택권을 갖고 AI 추천이 어디까지 개입하는지입니다. 아래 샘플은 실제 데모 캡처를 붙여 바로 부록 장표로 옮길 수 있는 구성입니다.

01 화면 1장

한 장표에는 대표 화면 하나만 크게 둡니다. 버튼 설명은 화면 옆에 번호로 붙입니다.

02 조작 3개

“누르면 무엇이 열림”, “무엇이 접힘”, “무엇이 바뀜”까지만 씁니다.

03 방어 문구 1개

추천은 자동결제가 아니라 확인/선택 레이어라는 점을 매 화면마다 짧게 남깁니다.

04 발표 멘트

상세 설명은 장표 본문이 아니라 발표자 멘트로 빼서 본문 밀도를 낮춥니다.

SAMPLE 01

메인 화면: Wallet을 열면 추천 조합이 결제 CTA 위에 먼저 노출됩니다.

메인 화면에서는 추천이 결제를 대체하지 않고, 기존 Samsung Wallet 결제 흐름 위에 얹히는 보조 레이어로 보이게 해야 합니다.

1 추천 보기

현재 매장과 보유 카드 기준의 추천 조합을 바텀시트로 엽니다.

2 다른 매장이에요

주변 매장 후보를 보여주고, 사용자가 선택 후 적용해야 추천이 갱신됩니다.

3 이 카드로 결제

추천 조합을 쓰지 않고 현재 카드로 바로 Samsung Pay 결제 흐름에 진입합니다.

장표 본문 추천 문구

“AI가 카드 결제를 자동 실행하는 것이 아니라, 결제 직전 선택지를 먼저 정리해주는 구조입니다.”

피해야 할 문구

“가장 좋은 카드로 자동 결제”, “혜택 확정”, “무조건 이득”처럼 단정적인 표현은 피합니다.

발표 멘트 예시: “사용자가 Wallet을 열었을 때 기존 결제 버튼은 그대로 두고, 그 위에 SWAP 추천을 얹었습니다. 그래서 사용자는 추천을 확인할 수도 있고, 무시하고 기존 카드로 결제할 수도 있습니다.”
SAMPLE 02

추천 멘트 접기: 사용자가 원하면 추천 영역은 작은 재활성화 상태로 축소됩니다.

추천이 계속 화면을 점유하면 결제 앱의 빠른 실행성이 떨어집니다. 접힘 상태는 추천 기능을 숨기되, 다시 켤 수 있는 진입점은 남깁니다.

1 위쪽 화살표

추천 배너를 접고 카드 캐러셀과 결제 CTA를 더 넓게 보여줍니다.

2 접힌 문구

“결제 추천 다시 켜기”처럼 기능 상태만 짧게 보여줍니다.

3 다시 펼치기

사용자가 누르면 원래 추천 배너와 매장 질문, 추천 보기 버튼이 복구됩니다.

상태 노출 정보 의도
펼침 매장명, 추천 보기, 다른 매장 선택 추천을 적극 확인할 수 있는 상태
접힘 Galaxy AI 로고, 재활성화 문구 결제 화면을 방해하지 않는 상태
발표 멘트 예시: “추천이 항상 강하게 떠 있으면 광고처럼 보일 수 있어서, 접기 버튼을 통해 사용자가 노출 강도를 직접 조절하게 했습니다.”
SAMPLE 03

추천 보기: 카드, 쿠폰, 포인트, 멤버십을 한 조합으로 묶어 보여줍니다.

사용자가 여러 앱을 오가지 않도록, 결제 전 필요한 항목을 계산대 행동 순서에 맞게 정리합니다.

1 조합 리스트

카드, 쿠폰, 포인트, 멤버십을 같은 화면에서 확인합니다.

2 예상 혜택 박스

결제 전에는 예상 또는 가능 상태로 보여주고, 확정 표현은 쓰지 않습니다.

3 이 조합으로 결제

사용자가 명시적으로 선택한 뒤 기존 결제 플로우로 넘어갑니다.

좋은 설명

“보유 카드와 등록 혜택 기준으로 예상 조합을 계산하고, 결제 전 사용자가 확인합니다.”

보완 포인트

청구할인, 실적 산입, 포인트 적립은 카드사 기준에 따라 사후 확인이 필요하다는 톤을 유지합니다.

발표 멘트 예시: “이 화면의 목적은 추천 이유를 길게 설명하는 것이 아니라, 계산대에서 바로 써야 하는 항목을 한 번에 정리하는 것입니다.”
SAMPLE 04

다른 기준 보기: 기준을 누르면 바로 바뀌지 않고, 변경 결과를 먼저 미리보기합니다.

추천 기준 변경은 사용자 통제권을 보여주는 핵심 장면입니다. “AI가 마음대로 바꾼다”는 인상을 줄이는 데 중요합니다.

1 혜택 / 마일리지 / 실적

사용자가 이번 결제에서 우선할 기준을 직접 선택합니다.

2 변경 미리보기

선택 기준에 따라 카드와 예상 결과가 어떻게 달라지는지 먼저 보여줍니다.

3 이 기준으로 보기

적용 버튼을 눌렀을 때만 추천 조합과 카드 선택이 바뀝니다.

기준 사용자 의미 장표 표현
혜택 이번 결제에서 예상 절감이 큰 조합 예상 혜택 우선
마일리지 할인보다 적립 가치를 우선 적립 가치 우선
실적 다음 달 혜택 조건에 가까워지는 카드 실적 관리 우선
발표 멘트 예시: “기준을 탭하는 순간 바로 결제수단을 바꾸지 않았습니다. 먼저 바뀔 결과를 보여주고, 사용자가 적용할 때만 반영되도록 설계했습니다.”
SAMPLE 05

다른 매장이에요: 위치 오차를 인정하고 사용자가 매장을 직접 고칠 수 있게 합니다.

복합몰, 지하상가, 역 주변에서는 위치 후보가 틀릴 수 있습니다. 이 장면은 기술적 한계를 숨기지 않고 UX로 방어하는 장면입니다.

1 주변 후보

현재 위치와 가까운 후보 매장을 리스트로 보여줍니다.

2 매장 선택

사용자가 실제 결제 매장을 고르면 추천 기준의 입력값이 바뀝니다.

3 이 매장으로 보기

적용 후에만 카드, 쿠폰, 멤버십 조합이 새 매장 기준으로 갱신됩니다.

좋은 설명

“기기 안에서 주변 매장 후보를 확인하고, 사용자가 실제 결제 매장을 선택합니다.”

피해야 할 설명

“정확한 매장을 실시간 추적합니다”처럼 상시 위치 추적처럼 들리는 표현은 피합니다.

발표 멘트 예시: “위치 인식은 편의를 위한 후보 제안이고, 최종 매장 선택은 사용자가 합니다. 이 구조가 신뢰와 통제권을 동시에 확보합니다.”
SAMPLE 06

결제 순서: 쿠폰 → 멤버십 → 카드 결제 순으로 계산대 행동을 쪼갭니다.

SWAP의 추천은 결제 전 의사결정뿐 아니라, 실제 계산대에서 어떤 순서로 보여줘야 하는지까지 정리합니다.

1 쿠폰 먼저

쿠폰이나 포인트처럼 결제 전 제시해야 하는 항목을 먼저 띄웁니다.

2 멤버십 적립

브랜드 멤버십이나 포인트 적립 바코드를 다음 단계로 보여줍니다.

3 카드 결제

마지막에 기존 Samsung Pay의 NFC 결제 흐름으로 이어집니다.

단계 사용자 행동 버튼 문구 예시
쿠폰 직원에게 쿠폰 바코드 제시 쿠폰 사용 완료
멤버십 멤버십 바코드 제시 멤버십 적립 완료
결제 리더기에 휴대폰 태그 결제 완료
발표 멘트 예시: “결제 앱 안에서 혜택을 보여주는 데서 끝나지 않고, 실제 매장에서 직원에게 무엇을 먼저 보여줘야 하는지까지 안내합니다.”

Appendix 장표 구성 초안

Appendix A. 메인 화면 작동 방식

  1. Wallet 진입 시 SWAP 추천 배너가 카드 결제 CTA 위에 노출
  2. 추천 보기, 다른 매장 선택, 기존 카드 결제의 세 가지 경로 제공
  3. 핵심 메시지: 추천은 선택 보조이며 기존 결제 UX를 대체하지 않음

Appendix B. 추천 노출 강도 조절

  1. 추천 배너를 접으면 카드 캐러셀과 결제 버튼 중심 화면으로 전환
  2. 접힌 상태에서도 재활성화 진입점 유지
  3. 핵심 메시지: 사용자가 추천 개입 수준을 직접 조절

Appendix C. 추천 조합 상세

  1. 카드, 쿠폰, 포인트, 멤버십을 한 조합으로 제시
  2. 예상 혜택과 추천 이유는 확정이 아닌 확인 가능한 정보로 표현
  3. 핵심 메시지: 앱 전환 없이 결제 전 판단 비용을 낮춤

Appendix D. 기준 변경과 위치 수정

  1. 혜택, 마일리지, 실적 기준을 미리보기 후 적용
  2. 매장 후보가 틀리면 사용자가 직접 선택 후 적용
  3. 핵심 메시지: AI 추천보다 사용자 통제권을 우선

Appendix E. 결제 실행 순서

  1. 쿠폰 바코드, 멤버십, NFC 결제 순으로 실제 행동 안내
  2. 각 단계 버튼은 “다음”보다 “사용 완료”처럼 행동 완료형으로 표기
  3. 핵심 메시지: 추천이 계산대에서 바로 실행 가능한 순서로 정리됨

Appendix F. 결과 화면과 안전 문구

  1. 결제 후 예상 혜택, 실적 상태, 확인 필요 항목 노출
  2. 청구할인/실적 산입은 카드사 앱에서 최종 확인 가능하다고 안내
  3. 핵심 메시지: 혜택을 과장하지 않고 사후 확인 루프를 제공
